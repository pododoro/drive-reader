# Branching Guide

## Main Branch

- `main` is the stable integration branch.
- Merge only reviewed and validated work into `main`.

## Branch Naming

Use short, descriptive lowercase names with hyphens.

Recommended prefixes:

- `feature/` for new functionality
- `fix/` for bug fixes
- `docs/` for documentation-only changes
- `chore/` for maintenance and tooling

Examples:

- `feature/naver-snapshot-save`
- `fix/blog-extraction-fallback`
- `docs/prd-update`
- `chore/daily-log-template`

## Pull Request Rules

- Open a PR for every meaningful change set.
- Keep each PR focused on one purpose.
- Prefer small PRs that are easy to review.
- Add a short summary, verification notes, and any follow-up risk.
